What Shipped

People can now help improve the dataset from anywhere.

  • A reviewer website is live. An invited reviewer can open a private link, learn the rules, label images, and come back later.
  • The work is organized. Every review is tied to a specific task, image, decision, and reviewer session.
  • We added a second model to compare. This helps us see whether a labeling problem belongs to one model or to the task itself.
  • Difficult images are easier to inspect. Zooms and detector suggestions help people find small or blurry trucks, but a person still makes the final call.

Why This Matters

The value is in the record we build over time.

flowchart LR
  A[Keep collecting] --> B[Organize the history]
  B --> C[Ask people to check hard cases]
  C --> D[Build a more useful dataset]
  D --> E[Train better tools later]
  E --> A

This week we improved the machinery for this loop. We did not produce a new model-accuracy result yet. The immediate value is a growing, reviewable record of how a port operates.

Human Review

The first outside reviewer has started using the system.

  • Simple for the reviewer. They open a private link, watch a short guide, and label the images.
  • Useful for PortMind. The system saves the decision and the evidence needed to understand it later.
  • Not a finished benchmark yet. The first packet still needs to be completed and checked before we use it to make a larger claim.

Next Step

Finish the proof before we scale the operation.

  • Finish the current review. The latest readout is 8 of 24 saved labels.
  • Add another independent reviewer. We need more than one person to check the hardest images.
  • Create a trusted test set. This will tell us where the models work and where they fail.
  • Then scale model training. We will spend more on models after we know the data can support a fair test.

The near-term goal is straightforward: turn a large stream of port data into a reliable asset that future software and models can use.
